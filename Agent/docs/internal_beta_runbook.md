# 内测运行说明

## 访问入口

- 内测地址：http://127.0.0.1:5003
- 聊天页：http://127.0.0.1:5003/chat
- 管理页：http://127.0.0.1:5003/admin

账号由管理员统一创建和发放。不要在群里公开管理员账号或密码。

## 当前建议内测范围

第一批建议控制在 3-5 人，优先测试以下场景：

1. 知识库问答
   - NAS/网盘地址和使用方式
   - 场地使用收费表、教室/会议室/报告厅收费
   - 常见制度、办法、流程查询

2. 公文生成
   - 简短通知
   - 工作报告初稿
   - 对策建议初稿
   - 已有材料改写为正式公文

3. 知识库管理
   - 上传文件
   - 删除文件
   - 查看向量库/表格入库状态
   - 检查操作日志

## 暂不建议测试

- 高度敏感的人事隐私材料
- 未公开合同全文
- 大量财务明细或个人身份信息
- 需要严格法律、财务结论的场景

如必须测试敏感材料，需先确认模型调用和数据范围。

## 已知观察点

- 问法不同但意思相同的问题，应观察是否回答一致。
- Excel 表格类问题，应核对金额、人数、日期等字段是否与原表一致。
- 公文生成应重点观察是否脑补事实，例如未提供的地点门牌、讲师、附件、调休规则。
- 引用来源应与答案内容相关，避免混入无关文件。

## 反馈格式

发现问题时请按以下格式记录：

```text
问题类型：
原始提问：
实际回答：
期望回答：
涉及文件：
是否可复现：
截图或备注：
```

```bash

```

```bash
pgrep -fl "python.*app.py --port 5003|monitor_knowledge_ops.py"
```
