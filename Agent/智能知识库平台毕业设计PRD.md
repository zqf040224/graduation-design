# 智能知识库平台毕业设计 PRD

## 1. 项目定位

本项目是一套面向团队知识管理与文档生成场景的智能知识库平台。系统通过本地知识库检索、聊天式问答、材料归纳、公文/文档生成、文件上传入库、后台管理和导出能力，帮助用户完成资料查询、内容整理和规范文档产出。

项目保留私有化部署和本地数据管理思路，默认不依赖云端存储。大模型仅用于文本理解与生成，知识库文件、运行记录和任务状态均由本地服务管理，适合作为毕业设计中的“智能知识库与文档生成系统”案例。

## 2. 核心目标

- 提供统一的 Web 入口，覆盖登录、聊天、知识库、上传、导出和后台管理。
- 支持本地资料入库、向量检索、结构化表格解析和来源追踪。
- 支持问答、摘要、改写、文档起草、文档格式化等常见办公辅助能力。
- 提供轻量后台任务机制，支撑异步入库、重建索引等耗时操作。
- 保持模块化架构，便于扩展新的聊天意图、工具服务和管理功能。

## 3. 用户角色

| 角色 | 主要能力 |
| --- | --- |
| 普通用户 | 登录系统、发起知识库问答、上传临时附件、生成和导出文档 |
| 管理员 | 管理用户、维护知识库文件、查看入库状态、触发重建索引、检查存储健康 |
| 系统维护者 | 配置模型密钥、存储路径、任务线程数、跨域策略和运行环境 |

## 4. 功能范围

### 4.1 智能聊天

- 通过 `/api/chat` 提供 SSE 流式响应。
- 使用意图识别将用户请求分派到知识问答、文档起草、格式化、表格处理等服务。
- 支持答案来源整理，避免暴露底层 chunk、路径、行号等内部检索坐标。

### 4.2 知识库管理

- 支持文档上传、解析、切分、向量化和入库。
- 支持文件状态查看、单文件重建索引、批量维护和审计记录。
- 支持结构化表格数据解析，用于数值类问答和 Excel 导出。

### 4.3 文档生成与导出

- 支持通知、报告、请示、说明等常见文档类型的生成。
- 支持 Word 导出、Excel 导出和模板化内容输出。
- 支持对生成内容进行审查、修订和格式化。

### 4.4 后台任务

- 使用 SQLite 保存任务状态。
- 使用本地线程池执行轻量异步任务。
- 通过 `/api/jobs/<job_id>` 查询任务状态。

## 5. 技术架构

```text
app.py
  -> create_app_context()
  -> register_routes(app, context)
  -> routes/*
  -> service layer
  -> chat runtime / knowledge base / export / job service
```

核心模块说明：

| 模块 | 职责 |
| --- | --- |
| `app.py` | Flask 应用入口、CORS/cache hook、路由注册和启动 |
| `app_dependencies.py` | 启动阶段依赖装配 |
| `app_context.py` | 运行期上下文和 service factory |
| `routes/` | HTTP 路由层，负责请求包装与认证 |
| `chat_runtime.py` | 聊天主链路与意图分派 |
| `chat_container.py` | 聊天服务懒加载装配 |
| `job_service.py` | 后台任务状态管理 |
| `upload_service.py` | 文件上传和知识库入库包装 |
| `agents/` | 文档生成、规划、检索、审查等 Agent 能力 |

## 6. 数据与安全

- 密钥通过 `.env` 本地配置，不提交到仓库。
- 原始知识库资料不纳入毕业设计代码仓库，可使用示例材料单独构建。
- 本地运行数据库、上传缓存、导出产物和向量索引均视为运行数据，不作为源码交付内容。
- 回答中应避免泄露底层检索路径、chunk 编号、表格行号等实现细节。

## 7. 验收标准

- 能启动 Flask 服务并访问登录页、聊天页、管理后台和健康检查接口。
- 能完成一次临时文件上传、一次知识库问答和一次文档导出。
- 管理后台能查看知识库文件状态并触发重建索引任务。
- 单元测试和核心模块编译检查通过。
- 代码与文档中不包含原始机构名称、真实人员、真实账号和真实业务材料。
