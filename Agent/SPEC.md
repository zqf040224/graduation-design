## 二、Agent 职责

### 1. Planner Agent (规划器)
```python
角色：任务指挥官
能力：
- 理解用户需求意图
- 分解任务为可执行步骤
- 判断是否需要联网搜索
- 协调其他 Agent 工作
- 决定是否需要多轮迭代
```

### 2. Search Agent (搜索器)
```python
角色：信息获取者
能力：
- 搜索最新政策、新闻
- 获取相关背景信息
- 验证事实准确性
- 过滤无效信息
```

### 3. Knowledge Agent (知识库)
```python
角色：规范顾问
能力：
- 检索公文格式规范
- 查找参考范文
- 提供格式指导
- 匹配合适的模板
```

### 4. Writer Agent (写作者)
```python
角色：公文起草者
能力：
- 按照规范生成公文
- 应用正确的格式
- 组织语言表达
```

### 5. Reviewer Agent (审查器)
```python
角色：质量把关
能力：
- 检查格式是否符合规范
- 验证内容逻辑
- 识别错误和遗漏
- 提供修改建议
- 给出置信度评分
```

## 三、工作流程示例

### 用户输入：
"帮我写一份关于抢抓深圳 AI 产业发展机遇的对策建议"

### Agent 执行流程：

```
Step 1: [Planner] 任务规划
├─ 识别任务类型：对策建议公文
├─ 判断需要联网：YES (需要最新 AI 产业动态)
├─ 规划步骤：
│   1. 联网搜索深圳 AI 产业最新政策
│   2. 检索对策建议格式规范
│   3. 生成公文草稿
│   4. 审查校验
└─ 分配任务给 Search Agent

Step 2: [Search] 联网搜索
├─ 搜索：深圳 AI 产业政策 2025
├─ 搜索：人工智能产业发展动态
├─ 整理关键信息：
│   • 深圳 AI 产业现状
│   • 国家政策支持
│   • 面临挑战
└─ 反馈给 Planner

Step 3: [Knowledge] 知识库检索
├─ 检索：对策建议格式规范
├─ 检索：优秀范文参考
└─ 反馈给 Writer

Step 4: [Writer] 生成草稿
├─ 综合搜索结果 + 格式规范
├─ 生成对策建议公文
└─ 提交审查

Step 5: [Reviewer] 审查校验
├─ 检查格式：✓ 符合规范
├─ 检查内容：✓ 逻辑完整
├─ 置信度：85%
└─ 如需改进，返回 Writer 重新生成

Step 6: 输出最终结果
```

## 五、技术实现

### 工具集
- Web Search API：搜索最新信息
- 知识库检索：TF-IDF/向量检索
- LLM：阿里云百炼 qwen3.5-plus
- 文档生成：python-docx

### 反馈机制
1. **自动纠错**：Reviewer 发现问题 → Writer 重新生成
2. **置信度评估**：每步输出置信度评分
3. **用户确认**：高风险任务需用户确认
4. **多版本生成**：可生成多个版本供选择
